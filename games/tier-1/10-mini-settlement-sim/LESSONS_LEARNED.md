# Level 10 Lessons Learned

* Daily resolution order is a gameplay rule, not an implementation detail.
* Failure before production creates meaningful planning pressure.
* A tiny scripted event can create strategy without becoming a full sim engine.
* Three citizens, three resources, and one event were enough to make a real settlement decision loop.
* The visible ledger made the simulation understandable without dev tools.
* Self-starting capture scripts are efficient, but should use strictPort or verify the spawned server URL to avoid ghost-server screenshots.
