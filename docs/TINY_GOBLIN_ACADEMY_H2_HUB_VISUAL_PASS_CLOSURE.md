# Tiny Goblin Academy H2 Hub Visual Pass Closure

## Status

H2 Visual/Icon Pipeline: Passed with Known Future Polish Debt

## Purpose

This document formally closes the H2 visual pipeline arc. 
* H2 successfully turned the hub from a rejected vertical list into a desktop-first game-grid dashboard.
* The hub remains strictly read-only, serving as a functional visual scaffold for the curriculum roster.
* The icon sheet pipeline was modernized and is now entirely manifest-driven.
* Fake checkerboard transparency was diagnosed and fixed through a derived transparent sheet without destructive manual cutouts.
* The asset intake pipeline has been formalized and standardized for all future visual assets.

## What Passed

* 2×5 grid layout exists and functions.
* The original vertical list has been rejected and completely replaced.
* A modal detail view exists for selected games.
* No launcher, install, or runtime behaviors were prematurely added.
* Source rectangles were successfully mapped for the irregular concept sheet.
* A derived transparent sheet is actively used to render flawless true-alpha icons over the UI.
* The original source sheet is preserved and untouched.
* No manual cutouts were required at any stage.
* All JSON validators pass gracefully.
* The Hub React/Vite build passes without warnings.

## Known Visual Debt

* The hub is vastly improved and accepted for Phase 1.5, but it is not final product art.
* Card frame spacing, padding, and layout scale may be further polished later.
* The modal layout structure can be improved later.
* The boot screen can be further branded/integrated later.
* No game launching exists yet (all buttons disabled).
* Level 1 (Button Goblin Clicker) source remains restoration-deferred.
* Future asset sheets must pass the formal intake pipeline before use.

## Doctrine Proven

* Generated asset sheets are source pantries, not ready-to-use runtime assets.
* Defining source rectangles via manifests prevents manual cutting and slicing.
* An alpha audit is mandatory for all ingested concept sheets.
* Fake checkerboard backgrounds may require programmatic derived cleanup.
* Animation sprite sheets require extra caution (pilot tests, contact-sheet review, frame manifests) to prevent ruined frames.
