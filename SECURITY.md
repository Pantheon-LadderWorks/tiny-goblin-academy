# Security Policy

Tiny Goblin Academy is currently an experimental, educational project and is not security-critical software. However, we still take security seriously.

### Reporting a Vulnerability

* **Do not report secrets or vulnerabilities publicly.**
* If you find a minor, non-sensitive bug, please open a standard GitHub Issue.
* For sensitive concerns (e.g., exposed credentials, major structural exploits), please wait until a direct contact channel is listed by the maintainers, or use the maintainer's preferred public channel if you are already in contact.
